# 开发协作规范

本项目遵循以下开发和提交规则。

## 分支规则

- 所有开发工作必须在开发分支上进行。
- 开发分支名称使用创建分支时的时间点，格式为 `y.M.dd.HHmm`，例如 `6.5.19.1003`。
- 分支名前面的年份只取公历年份最后一位，例如 2026 年写作 `6`。
- 阶段性成果完成后，将开发分支合并到 `main` 或 `master`。
- 当前仓库优先使用 `main` 作为主分支。

## 提交信息规则

创建开发分支后的首个提交必须包含：

```text
目标：
...

当前进度：
...

下一步计划：
...
```

同一开发分支后续提交不再写“目标”，但必须包含：

```text
当前进度：
...

下一步计划：
...
```

所有提交信息使用中文。

## 文档规则

- 项目必须维护持续更新的设计文档。
- 设计文档使用 `.docx` 格式，路径为 `docs/低延迟视频AI推理系统_设计文档.docx`。
- 设计文档需要覆盖架构流程图、模块图、开发方案、运行方式、后续计划等内容。
- 当架构、模块、接口或运行方式发生变化时，需要同步更新设计文档。

## 语言规则

- 项目实现尽量使用 C++。
- 如遇到性能验证、模型转换、工具链胶水、服务接口或生态问题，可以使用 Python 辅助实现。
